Updates

3.8.01 | 5/10/25
Core Update

Coreutils Rewritten in Rust: The entire base coreutils suite has been re-implemented in Rust to maximize both performance and memory safety. This includes replacements for critical shell-level tools such as ls, cat, cp, and mv. These new implementations are faster, safer, and more maintainable in the long term, setting a new foundation for system-level operations on Gecko OS.

Gecko UI Rendering Enhancements: Improvements were made to the desktop environment's Gecko engine integration. We've optimized UI rendering paths for smoother animation timing and increased frame stability under multi-window conditions. This also improves responsiveness on lower-end systems and virtual machines.

Bluetooth Security Patch: A critical patch was deployed to fix a vulnerability in the Bluetooth stack. The exploit, which allowed unauthorized proximity-based data injection during pairing mode, has been mitigated with stricter device authentication, encrypted handshake verification, and updated timeout logic for stale session handling.

Neofsys Command Introduced: A new command-line utility named neofsys has been added. It serves as a system info splash similar to neofetch, but tuned for Gecko OS with branding, package manager integration, and GPU/driver info baked in. Useful for debugging and system snapshots.

APK Package Manager Improvements: Improvements have been made to the APK package management subsystem. APK parsing now supports a broader range of Android manifest configurations, significantly improving compatibility with modern APKs. Additionally, sandboxing routines were upgraded to provide better isolation when launching Android apps on Gecko.
